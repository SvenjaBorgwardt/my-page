// kaleido — individual feedback on writingsince feb 2025 · in production

Kaleido is a tool for giving individual feedback on writing. It came out of a frustration I had no answer for: with hundreds of papers to read each week, proper individual feedback simply wasn't possible. I started vibe coding in early 2025, and this was my first big project.

A paper comes in. The name is stripped locally and replaced with a code. An anonymisation agent removes every remaining personal detail. A set of agents analyses the work along separate dimensions — structure, evidence, language, coherence, grammar, content. A final agent writes the feedback, addressed to the writer rather than about them.

"The first time I handed back Kaleido feedback, something happened I hadn't seen before. People came back with questions about their own essays. Not about the mark. About the essay itself."

Kaleido works because I trained the agents on specific types of tasks. A version that fits other subjects is on the roadmap.

I called it Kaleido because a kaleidoscope turns the same fragments into a different pattern every time you look. That's what I want feedback to do.

// compass — direction, not destinationsince jun 2025 · fine-tuning

Watching how people use AI, I keep noticing one thing: they take the path of least resistance. AI creates a specific failure of self-assessment — a writer produces a good output and reads the quality of the output as evidence of their own learning. Cognitive science calls this the illusion of fluency.

For something to be learned, it has to be slightly harder than easy — Bjork's desirable difficulties. Socratic agents annoy actual learners. I'm fine-tuning a smaller model to sit between the two poles.

learner$ "my code errors: for i in range(10) print(i)"
standard$ "you're missing a colon. here's the fix."
socratic$ "what is a for loop? what is syntax?"
compass$ "read the error message again. look at line 1."
A compass points the way. It doesn't hand you the map.

// ute — unkomplizierte theken-eingabenov 2025 · 1st @ bäko hackathon

A voice-first ordering system for bakery counters. Built in a weekend with three other women. The assistant keeps talking to the customer. UTE listens, transcribes, matches against the catalogue, builds the order silently. When the assistant repeats the order back — which she'd do anyway — that serves as confirmation. Nothing gets clicked.

What if the best way for an AI to be useful is to disappear?

UTE is also a common German name — we wanted a presence at the counter, not a system.
