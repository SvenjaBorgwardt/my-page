N° 01 · FEATURE

KALEIDO / feedback that lands

A TOOL FOR GIVING INDIVIDUAL FEEDBACK ON WRITING. came out of a frustration i had no answer for: with hundreds of papers each week, proper individual feedback simply wasn't possible.

a paper comes in. name stripped locally, replaced with a code. agents analyse along separate dimensions — structure, evidence, language, coherence, grammar, content. a final agent writes the feedback, addressed to the writer rather than about them.

"they came back with questions about their own essays. not about the mark — about the essay."

i called it kaleido because a kaleidoscope turns the same fragments into a different pattern every time you look. that's what i want feedback to do.

N° 02 · FEATURE

COMPASS / direction

AI creates a specific failure of self-assessment: a writer reads the quality of the output as evidence of their own learning. cognitive science calls it the illusion of fluency.

for something to be learned, it has to be slightly harder than easy — bjork's desirable difficulties. socratic agents annoy actual learners. i'm fine-tuning something between.

TRANSCRIPT // a coding bug
LEARNER:"my code errors."
STANDARD:"missing a colon."
SOCRATIC:"what is a for loop?"
WHAT I WANT:"read the error. look at line 1."
a compass points the way.
it doesn't hand you the map.

N° 03 · FEATURE · 1ST PLACE

UTE / the AI that
DISAPPEARS

a voice-first ordering system for bakery counters. built in a weekend at the BÄKO hackathon with three other women. we won first place.

the assistant keeps talking to the customer. UTE listens, transcribes, matches against the catalogue, builds the order silently. when the assistant repeats the order back — which she'd do anyway — that serves as confirmation. nothing gets clicked.

what if the best way for an AI to be useful is to disappear?

UTE — unkomplizierte theken-eingabe. a common german name. a presence, not a system.
